# Database ER Diagram

This entity-relationship diagram maps out the SQLite persistence layer (`investigations.db`), which acts as the memory core for the application, storing reports and linking them to historical queries.

```mermaid
erDiagram
    INVESTIGATIONS {
        string id PK "UUID format"
        string type "Investigation category (website, logs, threat, question)"
        string user_input "Raw string provided by user"
        string input_summary "Truncated summary of input (max 100 chars)"
        string report "Full markdown generated by CrewAI"
        string severity "Extracted threat level (Critical, High, Medium, Low)"
        datetime timestamp "ISO-8601 creation time"
    }
    
    SYSTEM_LOGS {
        integer log_id PK "Auto-increment ID"
        string level "INFO, WARNING, ERROR"
        string message "System event description"
        datetime timestamp "Event execution time"
    }

    AGENT_MEMORY {
        integer memory_id PK "Auto-increment ID"
        string investigation_id FK "Links to INVESTIGATIONS.id"
        string agent_name "Which agent stored this (e.g., ReconAgent)"
        string key "Context key (e.g., extracted_ip)"
        string value "Context value (e.g., 192.168.1.1)"
    }

    INVESTIGATIONS ||--o{ AGENT_MEMORY : "generates context"
```
